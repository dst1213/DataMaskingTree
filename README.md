# DataMaskingTree
数据脱敏技术研究

<pre>
数据脱敏遵循的两个原则
      1）尽可能的为脱敏后的应用，保留数据脱敏前的有意义的信息
      2）最大程度防止黑客精心破解
</pre>

<pre>
常用的数据脱敏处理方法
      1）替换
            如统一将女性用户名替换为F，易破解
      2）重排
            序号12345重排为54321,易破解
      3）加密
            安全程序取决于加密算法
      5）截断
            13812345678截断为138对生产不够友好
      6）掩码
            123456掩码为1xxxx6
      7）日期偏移取整
            20130520 12:30:45   取整为   20130520 12:00:00
</pre>
